# DAG_practice
Практические DAG-файлы для Apache Airflow.  
Реализуют ETL-процессы: выгрузка данных из S3, трансформация, обогащение через справочники (PostgreSQL/Greenplum) и запись результатов обратно в S3.

## Описание DAG'ов

### 1. `etl1_t1` – Получение последнего файла из S3
- **Задача**: найти самый свежий CSV-файл в публичном бакете за текущую дату, скачать его и вывести в лог первые строки.
- **Особенности**: используется `S3Hook.list_keys` и `max()` для выбора последнего файла.

### 2. `etl1_t2` – Копирование последнего файла с преобразованием
- **Задача**: взять последний CSV-файл за день, преобразовать в Parquet (через `pandas`) и сохранить в личный бакет `culab-student-files` в папку `avia/YYYY-MM-DD/`.
- **Ключевые операторы**: `S3Hook.download_file`, `pd.read_csv`, `df.to_parquet`, загрузка в S3 через `load_bytes`.

### 3. `etl1_t3` – Загрузка всех файлов дня
- **Задача**: скачать **все** CSV-файлы за текущую дату, объединить в один DataFrame, сохранить единым Parquet-файлом в `avia_daily/avia_YYYYMMDD.snappy.parquet`.
- **Применение**: обработка потоковых данных, когда за день приходит несколько мелких файлов.

### 4. `etl2_t1` – Выгрузка из Greenplum
- **Задача**: подключиться к Greenplum, выполнить агрегирующий запрос по рейсам (группировка по дате, аэропорту, статусу). Результат логируется, но не сохраняется.
- **Демонстрация**: использование `PostgresHook` и параметризация запроса через `data_interval_start`/`end`.

### 5. `etl2_t2` – Обогащение данными аэропортов
- **Задача**: объединить агрегат рейсов из Greenplum со справочником аэропортов из PostgreSQL (`airports_full`). Итоговый DataFrame содержит название, тип, регион аэропорта.
- **Цель**: показать join данных из разных источников (GP + PG) в памяти Airflow.

### 6. `etl2_t3` – Полный пайплайн с записью в S3
- **Задача**: выполнить запрос к Greenplum, обогатить справочником аэропортов (Postgres), сохранить результат в S3 (Parquet) с партиционированием по месяцу и именованием файла по дате.
- **Формат вывода**: `users/<логин>/flight_status_mart/YYYYMM/status_YYYYMMDD.parquet`.
- **Используется**: `S3Hook.load_bytes` с буфером `io.BytesIO`.
